0001 The type contract

Callouts — distinct by role

The rule

Effect<A, E, R> means: requires R, can fail with E, can succeed with A.

Pitfall

Do not mentally translate Effect to “better Promise.” It is a typed executable plan.

Good rule

If it can fail, need a service, retry, time out, or be traced, keep it an Effect.

Wrong / right

trap
const p = fetch(url)
Effect.promise(() => p)
// already started
correct
Effect.promise(() => fetch(url))
// fires only on run
